## Findings considered and rejected

- **`/api/profile` mass-assignment / prototype-pollution risk**: the PUT
  handler iterates `Object.keys(current)` (a fixed `CompanyProfile`) and
  copies only string values — an effective whitelist. Not exploitable as
  written.
- **Error-detail leakage to clients**: `handle()` in `routes.ts` already
  returns a generic "Interner Fehler." for non-validation errors;
  server-side `console.error` is appropriate for a local tool.
- **Unvalidated `as Partial<VehicleInput>` cast on POST /api/vehicles**:
  cosmetic — `createVehicle` validates model/plate/klass/status at runtime
  (`vehicles.ts:185-191`).
- **`vite.config.ts` as dead config**: intentional, documented shim so the
  shadcn CLI resolves the `@/*` alias. Leave it.
- **Missing security headers (CSP, X-Frame-Options)**: low value for a
  local single-user tool; revisit only if the app is ever deployed (see the
  README note added by plan 003).
- **Buchhaltung column-definition memoization**: child tables aren't
  memoized, so memoizing columns alone changes nothing measurable. Revisit
  only as part of a (currently unjustified) Buchhaltung decomposition.
- **Fetch caching / request dedup / optimistic updates in hooks**: datasets
  are tiny and local; full-refetch-after-mutation is simple and correct.
  `src/lib/api.ts` (plan 005) is the home if this ever becomes worth doing.
- **DATEV import (counterpart to export)**: one-way export to the
  Steuerberater is the standard DATEV workflow; reimport adds GoBD/
  compliance risk for little value.
- **Frontend DOM test infrastructure (testing-library etc.)**: deferred —
  the highest-risk untested layer was the server (plan 004 covers it);
  adding a DOM test stack to a Bun-only repo is a larger decision than its
  current payoff.
- **`toast` missing from a useEffect dependency array (Profil.tsx)**:
  sonner's `toast` is a module-level singleton; not a bug.
- **Buchhaltung/Kalendar/Profil monolith decomposition**: real but
  expensive and risky relative to payoff while features are still moving
  fast; the calendar's seams get partially addressed by plan 009. Revisit
  after 009 lands.
